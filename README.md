# Проект "Магазин обуви" - ShoeStore

## О проекте
Комплексная система для управления интернет-магазином обуви, состоящая из нескольких компонентов:

- **Библиотека классов** (ShoeStoreDb) - слой доступа к данным и модели
- **Веб-приложение** (ShoeStoreWeb) - ASP.NET Core приложение с интерфейсом для пользователей
- **Десктопное приложение** (ShoeStoreWPF) - WPF приложение для администраторов
- **Веб-API** (ShoeStore.WebApi) - REST API для взаимодействия с системой

## Известные проблемы

### 1. Проблема с изображениями товаров
- Картинки не соответствуют продукту из-за некорректных данных в базе
- В текущей реализации изображения хранятся как VARBINARY в БД, что не соответствует требованию задания

### 2. Не реализовано хранение путей к изображениям
- По заданию требуется хранить путь к изображению в БД, а сами файлы сохранять в папке приложения
- Текущая реализация хранит изображения как байты в колонке ImageData

## Состав решения

### ShoeStoreDb (Библиотека классов)
Слой данных с моделями Entity Framework Core и контекстом базы данных.

### ShoeStore.WebApi (Веб-API)
REST API с JWT аутентификацией, контроллерами товаров и заказов.

### ShoeStoreWeb (Веб-приложение)
Пользовательский интерфейс с каталогом товаров, фильтрацией и системой заказов.

### ShoeStoreWPF (Десктопное приложение)
Административный интерфейс для управления товарами с возможностью загрузки изображений.

## Функционал

### Веб-приложение для пользователей
- Каталог товаров с фильтрацией
- Создание и просмотр заказов
- Авторизация по ролям (Клиент/Менеджер/Администратор)

### Десктопное приложение для администраторов
- Управление товарами (добавление/редактирование/удаление)
- Загрузка изображений товаров
- Расширенные возможности фильтрации

### Веб-API
- RESTful API для интеграции
- JWT аутентификация
- Управление товарами и заказами

## Технологии
- ASP.NET Core 8, Entity Framework Core
- Razor Pages, Bootstrap 5
- WPF, .NET 8
- SQL Server
- JWT и Cookie-based аутентификация

## Подготовка к запуску

### 1. Создание базы данных
Перед запуском необходимо выполнить SQL скрипт для создания базы данных ShoeStore и всех таблиц.

### 2. Настройка строки подключения
В файлах конфигурации каждого проекта нужно указать правильную строку подключения к SQL Server
